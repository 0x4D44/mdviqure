# Coverage Improvement Plan - 2026.01.04

**Goal:** Increase code coverage to >98% by refactoring for testability and adding comprehensive tests.

## Strategy

The primary strategy is **Dependency Injection**. We will decouple the application logic from the actual execution of system commands (`ffmpeg`, `ffprobe`). This allows us to unit test the orchestration logic by mocking the external environment.

## Stages

### Stage 1: Core Refactoring & Abstraction
1.  **Define `VideoTool` Trait**: Create a trait that abstracts the operations:
    *   `get_duration(input: &str) -> Result<f64>`
    *   `run_ffmpeg(args: &[&str]) -> Result<()>`
2.  **Implement Real `VideoTool`**: Create a struct (e.g., `FfmpegTool`) that implements this trait using `std::process::Command`.
3.  **Refactor Functions**: Update `reduce_video` and `get_video_duration` (or merge logic) to accept `&impl VideoTool`.
4.  **Extract `run`**: Create a `run(args: Args, tool: &impl VideoTool)` function to separate CLI parsing from execution.

### Stage 2: Mocking & Unit Tests
1.  **Create `MockVideoTool`**: A struct for testing that:
    *   Returns predefined durations for `get_duration`.
    *   Captures arguments passed to `run_ffmpeg` for verification.
2.  **Test `reduce_video`**:
    *   Verify it calculates the correct bitrate based on the mocked duration.
    *   Verify it calls `run_ffmpeg` with the correct arguments (bitrate, codecs, paths).
3.  **Test Argument Logic**:
    *   Test `run` with valid sizes (50, 100).
    *   Test `run` with invalid sizes (verify error returned).

### Stage 3: Verification
1.  Run `cargo llvm-cov` to verify coverage.
2.  Ensure all paths (success, failure, validation) are covered.

## Timeline
*   **Stage 1 & 2**: Immediate implementation.
*   **Stage 3**: Final verification.
