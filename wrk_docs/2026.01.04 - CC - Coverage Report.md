## Summary

The core business logic of calculating bitrates is well-tested. However, the application's primary function—interacting with `ffmpeg` and `ffprobe`—is completely untested. The lack of abstraction for system commands prevents unit testing of the workflow.

## Recommendations

1.  **Refactor for Testability**: Abstract the `Command` execution into a trait (e.g., `VideoProcessor` or `CommandRunner`).
2.  **Dependency Injection**: Pass this trait into the core logic functions (`reduce_video`, `get_video_duration`).
3.  **Mocking**: Implement a mock version of the trait for tests to simulate `ffprobe` output and verify `ffmpeg` command arguments.
4.  **Integration Tests**: Add high-level tests for the `run` logic.
